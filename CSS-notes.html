Common ways of writing CSS

  1. In-line CSS
  2. Internal CSS
  3. external CSS

In-line CSS

To style html inline, we simply use the style attribute on any of our html tags e.g.,
<h1 style="color:blue">heading is now blue</h1>

In line styles should never be used!

Internal CSS

To use internal styling, we add a style tag within the head tag e.g.,
<head>
 <style>
  color: blue;
 </style>
</head>

This will then apply a blue color to all h1 tags within our html. This approach seperates our concernes a bit and is a reccommened approach over In-line CSS but still not the best.

External CSS

The preffered approach is to create an external style sheet. We do this by creating a new file called say style.css.

Once we have created our stylesheet, we then need to link the css file to our html doc, we do this in the head tag. We do this with a link tag.

<head>
 <link href="./style.css" rel="stylesheet" />
</head>

Now this style sheet will be applied to our html document.

CSS properties

We always have a selector which determines where to apply the style e.g., if we have h1 {...} this will select all h1 tags and apply this styling to them.

Some other properties we can apply are

  • font-size: 26px; ←Sets the font size to 26px
  • font-family: sans-serif; ←Sets the font family
  • text-transform: uppercase; ←Allows us to transform the text in certain ways such as uppercase.
  • font-style: italic; ←Sets the font to italic, we could also say bold etc.

There is also a line height property which specifies the line height in relation to the size of the text, 1 = same size as text, 1.5 = 150% of text height.

The strong, em and a tags all inherit from a p tag which means that when we apply styling to a p tag, it will automatically be applied to the given tags, it will not override the html functionality however, e.g., a strong tag will always remain strong regardless of CSS.

Another CSS property is the text align which aligns the text on the vertical axis, e.g., left, center right. We would say
 text-align: centre;

Combining selectors

We have just applied font-family to the h1, h2, h3, h4, p and li tags. We can do this in a much better way, we do this by selecting many elements at once.

h1, h2, h3, h4, p, li {
 font-family: sans-serif;
}

This is called a list selector and selects all instances of the list.

Descendent selector

We can sytle an element which is within another element e.g.,
<footer>
 <p>This will be styled</p>
</footer>

The above will be styled below.

footer p {
 font-size: 16px;
}

So all p tags who are children of footer tags will be styled accordingly. This is not overly robust as its dependent on the html structure. This is not scalable.

Using Classes and ID for styling

We can set the id attribute to html tags then set styles based on that. e.g.,

<p id="author">James Jenkins</p>

We then select id's in CSS by using the # symbol e.g.,

#author {
 font-style: italic;
}

The '#' select id's, the '.' selects classes.

Classes vs Id's

The main difference between a class and an Id is that we cannot repeat an id, e.g., if we have an id of 'copywrite', we can only have one of those, if that were say a class, we could have many of them.

Setting class can be done like below.

.related-author {
 font-size: 18px;
 font-weight: bold;
}

Generally in the real world, we never use id's, we only use classes as this allows us to scale the page easily.

Something to note is that currently, some of out p tags are being selected up to three times. We will revisit this later.

Colors

In CSS there are a ref diferent ways we can represent colors, the two main methods are rgb or hex.

To use rgb notation we use say rgb(0, 255, 255), we can also use reba which allows us to specify the transperency of color, 0 fully transparent, 1 fully opaque. rgba(0, 255, 255, 0.3)

to use hex colors we just specify #xxxxxx, rgb in hexidecimal #00ffff this is the same as rgb(0, 255, 255).

When we have all the numbers exactly the same we get a black, white or gray color.

We specify the color of an element using the color property. e.g.,

h1 {
 color: #1098AD;
}

Borders

Borders are another property which we can style with CSS, it is called a short hand property as we can define many things at once. Generally it goes border: <width> <line fill> <color>

So border: 5px, solid, #1098AD; Would create a 5px solid border of color #1098AD. Other styles we can use are dotted, dashed etc.,

There are special variants such as border-top or border-bottom which will just style the top or bottom.

Pseudo Code

If we wanted to select the first element of a list to make it bold, the way we would go about it now is to hand the first element a class of say 'first-element', we would then write some CSS to select that class and make it bold. A much better way of doing this is to use CSS pseudo classes. This means we can select an element then put a colon and select other things specific things in relation to that. For example we could choose li:first-child which will get the first child from a list and style it. To make the first child bold we would say

li:first-child{
 font-weight: bold;
}


Some other examples are li:last-child{...}, li:nth-child(2){...} <- This one selects the second list element and styles it.
li:nth-child(odd) {...} <- styles every odd element. This one can be very good for styling tables which have alternating colors.


Based on what we have just learned, we would expect that we can now select the first p tag within our article using a pseudo class as follows
 article p:first-child{...}

However we will find this will not work, as first child will try to select the first child in the article tag, which in our case is the header tag. This would work if we had a paragraph as the first child under our article tag. Hencem pseudo classes dont work well for random classes within classes, but it works really well for things like lists and tables, this is because all children are often the same.

Styling anchors

We may try to style an anchor using a {...}, this is bad practce though as we can no longer style it based on it's state. A better practice is to us anchor pseudo classes.

So we may style them like below
a:link {
 color: #1095ad
}
This will ensure only a tags with links will by styled as opposed to all a tags regardless of whether it has a href or not.

We also have a visited pseudo class for visited links, we generally set this the same as the normal unvisited link and group them together.

A very useful pesudo class is the a:hover{...}, this will change the styling of the html whenever we hover over the link.

We also have a:active which is the styling for when we are clicking on the a tag.

We should always add anchor styling in this order, link, visited, hover, active.

.author {

 font-size: 18px;

#author-text {
 font-size: 20px;

p, li {

 font-size:22px

The html is being selected three times by CSS, which styling does it use? The answer is all of them.

The main issue is that there is conflicting font size decorations.

CSS has a list of priorities, the basic list is ID, class or pseudo class, general tags. So if we have a font size set for the tag of 19px and a font size set for the ID of 25px, the font will be 25px as the ID selector has higher priority than the tag selector.

If we have say two conflicting id selectors, its the last style in the document which will be used.

There is also a universal selector '*' but more on this later.

So in the above example, the CSS applied would be
{
 font-size: 20px;
 font-style: italic;
 font-family: sans-serif;
 color: #444l
}

In line styles have a higher priority than ID selectors and important styles (marked with a '!') have priority over all others.

  1. Important styles (!)
  2. Inline styles
  3. ID selector (#)
  4. Class Selector (.)
  5. Element selector
  6. Universal selector (*)

We can make a CSS style important as below.

footer p {
 color: green !important;
}

Inheritance and the universal selector

Inheritance comes from parent and children, for example we have a body tag as the parent then eveything inside that body tag is a child tag and inherits styles from it. For example, if we set the body tag to have a color of red, all tags within that body tag will be given a color of red, this is easily overridden however. It's almost like a base style for our page to fall back on.

There are a lot of styles which dont get passed down, one of these include the border, for example, if we set the body tag with a border-top: 10px solid #1095ad, that style will only be applied to the body tag and will not be passed down.

The properties which do get inherited are:

  • font-family
  • font-size
  • font-weight
  • font-style
  • color
  • line-height
  • letter-spacing
  • text-align
  • text-transformation
  • text-shadow
  • line-style
  • Others

If we would like everything to get selected, we could use the universal selector e.g., * {...}, we could then give it a property of say border-top: 10px solid white and it will give every element in out html a top border.

The Box Model

The box model is used in CSS and basically everything that is on a webpage is in a box with a given width and height. Around that content we have padding, then a border then margin. We can add any of these to a tag by specifying say padding-top, margin-bottom etc. A short hand way to add them is to just say padding: 10px 20px; the first number will be applied to the top and bottom, the second number will be applied to either side.

Collaping margins

Say we have the below...

<p class="e1">Element 1</p>
<p class="e2">Element 2</p>

.e1 {
 margin-bottom: 50px; }

.e2 {
 margin-top: 60px;
}

We may assume that we would add both margins together and get a 110px gap between the two elements. This is not the case, if we have two margins which are next to each other, CSS will select the larger of the two and just use that as spacing between the two items.

Dimentions

We can easily set the dimentions of elements usign the width and height properties. We never want to do this in HTML but rather in CSS. When we set one the other will be automatically assigned to keep the aspect ration. We can force a certain aspect ratio by setting both height and width.

We dont always have to use px to set the width and height, we can also use %, if we say 100% the element will take up the entire content area of the parent. So if we had an image and set the width to 100% it would stretch across the enitre width of the page keeping a consistent aspect ration. This allows us to begin creating a responsive web page.

Alignment and Centre

It is currently quite difficult to centre all the content on the page but there is a simple trick we can use to do so. First we wrap out entire content in a div tag, this is simply a tag which groups things together but has not semantic meaning. We then apply a class called content. We then style this content to have a width of say 700px and then have margin left and right to be automatic. This will keep all the content of the page at a max width of 700px then apply an automatic margin either side of the contet to keep the 700px contianer centred.
